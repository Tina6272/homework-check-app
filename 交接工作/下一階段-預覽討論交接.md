1. 讀取 `AGENTS.md`、本交接檔、Obsidian 駕駛艙。
2. 檢查 Git 狀態。
3. 若預覽失效，使用本機暫存副本重新啟動，不要在 Google Drive 專案資料夾安裝 `node_modules`。
4. 從學生端畫面開始討論，不先重做架構、不先改資料庫。

## 下一階段要討論的重點

1. 學生端畫面是否符合「學生自主完成作業」的核心用途。
2. 學生端流程：選作業、回報完成、訂正、送出狀態，哪些文字與按鈕需要調整。
3. 老師端畫面：目前需要先看哪些資訊，哪些資訊可以延後。
4. 家長端畫面：是否只做提醒與進度查看，避免變成家長代替學生管理。
5. 三端之間的資料流與狀態名稱，先用討論定案，再接 Firestore。
